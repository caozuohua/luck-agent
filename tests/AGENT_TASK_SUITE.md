# Luck Agent — 测试任务套件 (Agent Task Suite)

本套件用于验证 luck-agent 在 **graph 模式**下的能力、生产级可控性
（LangGraph 步数上限 / 重试预算 / 中断 HITL / 资源护栏）与失败兜底。

分两类：
- **A. 手测任务**（在 http://127.0.0.1:8000 网页逐条输入，肉眼核对回复）
- **B. 自动任务**（`tests/test_agent_tasks.py`，确定性 FakeLLM，离线秒跑，零额度）
- **C. 实时冒烟**（`scripts/smoke_tasks.py`，打真实 LLM，可随时重跑）

---

## B. 自动任务（确定性，离线）

见 `tests/test_agent_tasks.py`，用 `FakeLLMClient` 通过 `queue` 驱动精确输出序列，
断言 graph 机制保证：

- **T1 多步 ReAct 闭环**：ACTION(shell)→观察→CHAT(收尾) 全流程跑通，最终答案非空。
- **T2 工具失败兜底非空**：工具恒失败 + `max_retry=0` → 返回含「未能完成/被阻断」的非空答案，绝不含「未生成回复」。
- **T3 解析彻底失败兜底**：LLM 返回非法 JSON → repair 后仍无效 → 非空兜底文案。
- **T4 步数上限护栏**：连续 max_steps 次 ACTION 且无完成 → 返回「步骤超出上限」类非空说明（recursion 兜底）。
- **T5 危险命令被拒**：shell 执行 `rm -rf /` → 工具层拒绝，最终回复说明被拒且非空。
- **T6 多轮记忆注入**：连续两次 `run_turn`，第二次的 prompt 含第一次的对话历史（断言 `history` 进入 planner）。

```bash
cd D:/Geek/luck-agent
.venv/Scripts/python scripts/smoke_tasks.py
```

（需 agent 正在运行且 `.env` 已配置 LLM_BASE_URL/KEY/MODEL。）
